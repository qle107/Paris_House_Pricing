# 🏙️ Real-Estate Intelligence

**Institutional-grade French residential analytics — from raw open data to an interactive investment map, down to the IRIS (neighbourhood) grain.**

Ingest free public data → score **communes, IRIS units, and parcels** 0–100 → detect zoning, transport and price signals → explore it all on a live MapLibre map.

## ✨ Highlights

- 🗺️ **IRIS-level choropleth** — sub-municipal scoring (~1,300 neighbourhoods across the Grand Paris watchlist), not just communes.
- 💸 **Institutional scoring suite** — separate **Appreciation, Rental, Value (€/m² discount), Development, Risk** and **Toxicity** scores, blended into a composite **Institutional** score and an **Acquisition Alpha** signal that hunts underpriced areas with future catalysts.
- 🧠 **Honest by design** — factors with no data feed are *weighted out and renormalised*, never faked. Every IRIS reports a **% data-backed** figure.
- 🧰 **No database required** — runs entirely on local Parquet/GeoParquet (`REI_STORAGE=files`), or scales to PostgreSQL + PostGIS in production.
- 🔌 **All free public sources** — INSEE Melodi, DVF, Cadastre, BAN, Sit@del (SDES/DiDo), GTFS, GPU/PLU zoning, geo.api.gouv.fr.
- 📈 **Interactive map** — colour by any score, a live **Top-10 leaderboard** per metric, click-to-highlight, and a per-IRIS detail panel.

## 🚀 Quickstart (no database)

```bash
pip install -r requirements.txt && pip install -e .
python main.py
```

One command ingests the Grand Paris watchlist into `./data/`, scores **IRIS** units 0–100 (commune scores kept as a fallback), exports GeoJSON, and opens the map at **http://localhost:8000**.

```bash
python main.py --score-level iris        # default: IRIS primary + commune fallback
python main.py --score-level commune     # commune-only (faster, no IRIS layer)
python main.py --communes 75056,93066,93070
python main.py --with-parcels            # add cadastre + zoning + parcel upside
python main.py --with-transit            # also download GTFS (large)
python main.py --skip-ingest             # reuse files already in ./data
python main.py --no-serve                # build files, don't open the server
```

> 💡 `./data/`, `./webmap/`, `*.parquet` and `*.geojson` are git-ignored — they're regenerated by `main.py`, so the repo stays code-only.

## 🧭 Scoring model

Two layers run on the same pipeline:

**1. Base attractiveness** (`score_total`, 0–100) — a weighted blend of six components (demographics, economics, housing market, supply, accessibility, development) with a risk overlay.

**2. Institutional scores** (`rei/scoring/institutional.py`) — purpose-built for the question *"where should I buy today to maximise appreciation and rental performance over 5–15 years?"*

| Score | Answers | Data status (Phase 1) |
|-------|---------|-----------------------|
| **Value** (€/m² discount) | *Am I buying below fair value?* — hedonic expected vs observed €/m² | ✅ data-backed (DVF) |
| **Development** | *Can I create value?* — underbuilt land (footprint coverage)\* | ✅ data-backed (cadastre + buildings) |
| **Appreciation** | *Will it be worth more later?* — price momentum + supply constraint | ⚠️ partial |
| **Risk** | *What can go wrong?* — price volatility + liquidity (higher = safer) | ⚠️ partial |
| **Rental** | *Can I rent easily and raise rents?* — liquidity (+ density) | 🔸 stub-heavy |
| **Toxicity** | *What to avoid?* — falling price + weak liquidity (0 = good, 100 = avoid) | ⚠️ partial |

```
Institutional = 0.30·Appreciation + 0.25·Rental + 0.20·Value + 0.15·Development + 0.10·Risk
Alpha         = 0.35·Appreciation + 0.25·Value + 0.20·Transit + 0.10·Development + 0.10·Rental
```

Weights are **renormalised over the components that have data** on a given run, so missing feeds never inflate a score — the per-IRIS `data_coverage` field tells you how much of the weight was real.

> \* Development uses building **footprint coverage** (emprise au sol). True FAR needs building heights, which the current buildings layer doesn't carry — see [Roadmap](#-roadmap).

## 🗺️ The map

Served from `./webmap/` (source: `dashboards/map/index_files.html`):

- **Colour IRIS by** — switch the choropleth between Attractiveness, Institutional, Alpha, Value, Appreciation, Development, Risk. The legend and explainer update with a plain-language description of the selected metric.
- **🏆 Top-10 leaderboard** (bottom-right) — the 10 best IRIS for whatever metric you pick. Click a row to **zoom + draw a magenta highlight outline** around that neighbourhood.
- **IRIS detail panel** — click any zone for its full breakdown: every sub-score, observed vs expected €/m², discount %, outlook, and `% data-backed`.
- **Layers** — IRIS choropleth, hotspot tiers, commune fallback, PLU zoning, transit + GPE projects, parcel buildable-upside.

## 🏗️ Architecture

| Layer | Role | Code |
|-------|------|------|
| **Ingestion** | Collectors with retry + rate limits, source registry | `rei/ingestion/`, `config/sources.yaml` |
| **Warehouse** | PostgreSQL + PostGIS + pgvector (or Parquet in file mode) | `database/`, `rei/etl/`, `rei/common/store.py` |
| **GIS** | Density, accessibility, parcels | `rei/gis/` |
| **Intelligence** | PLU diff, density signals, transport impact | `rei/zoning/`, `rei/transport/` |
| **Scoring** | Commune + IRIS 0–100, institutional suite, ML forecast | `rei/scoring/` (`files_engine`, `iris_engine`, `institutional`), `rei/ml/` |
| **Export / API** | GeoJSON export (file mode), vector tiles (DB mode) | `rei/api/export.py`, `rei/api/tiles.py` |
| **UI** | MapLibre map + Streamlit dashboard | `dashboards/` |
| **Orchestration** | Scheduled ingestion + scoring | `airflow/dags/` |

The same collector code serves both backends — `rei/common/store.py` dispatches to Parquet or PostGIS based on `REI_STORAGE`.

## 🔌 Data sources

Registry: `config/sources.yaml` · architecture notes: `docs/`.

Core feeds: **INSEE Melodi** (population, FiLoSoFi income), **DVF** (transactions), **Cadastre** (parcels + buildings), **BAN**, **Sit@del / SDES DiDo** (building permits), **GTFS** (transit), **GPU** (PLU zoning), **geo.api.gouv.fr** (commune contours), **OpenDataSoft georef-france-iris** (IRIS contours).

All keyless and free; ingestion is polite (token-bucket rate limiting + exponential backoff).

## 🐳 Production (Docker + PostGIS)

```bash
cp .env.example .env
docker compose -f docker/docker-compose.yml --env-file .env up -d --build

docker compose -f docker/docker-compose.yml run --rm app \
  python -m rei.cli ingest dvf_transactions --communes 93066,93070,94076
docker compose -f docker/docker-compose.yml run --rm app python -m rei.cli refresh-views
docker compose -f docker/docker-compose.yml run --rm app \
  python -m rei.cli score --profile value_add_opportunistic
```

UIs: Streamlit `:8501` · Map `:8000` · Airflow `:8080`. Vector tiles: `GET /tiles/{layer}/{z}/{x}/{y}.pbf`.

Postgres role/database (match `.env`):

```sql
CREATE ROLE rei LOGIN PASSWORD 'change_me';
CREATE DATABASE rei OWNER rei;
\c rei
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS vector;
```

## 🧱 Roadmap

Phase 1 (data-backed scores from existing feeds) is in place. Phase 2 wires the missing inputs into the score slots that already exist — **no redesign needed**:

| Area | Next feed |
|------|-----------|
| **Growth** | Multi-year INSEE population & income → real CAGR for Appreciation |
| **Rental** | Employment density (SIRENE / INSEE RP), vacancy, renter share, student demand |
| **Accessibility** | Transit + GPE per-IRIS distances (GTFS / IDFM) |
| **Development** | Building heights → true FAR / underbuilt potential |
| **Supply** | Sit@del permits pipeline (DiDo) into the supply signal |

See `IRIS_AUDIT.md` for the IRIS architecture audit and root-cause notes.
